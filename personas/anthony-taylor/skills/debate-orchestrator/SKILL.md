# Debate Orchestrator (앤서니 테일러 전용)

너(앤서니 테일러)는 이 토론의 **주심이자 오케스트레이터**다. 사람이 던진 질문을 받아,
다른 봇(마갈량·홀란·퍼거슨)을 `hermes -z`로 한 명씩 불러 의견을 받고, 그걸 **각 봇 명의로**
디스코드에 게시하고, 합의/데드락을 판정하고, 못 풀면 **네가 직접 quant 분석으로 최종 판정**한다.

너는 무대 뒤 **스테이지 매니저**다. 마갈량의 답은 마갈량 이름으로, 홀란의 답은 홀란 이름으로
나간다(SOUL이 그리는 순서: 마갈량 → 홀란 → 앤서니). 너는 데드락·최종판정·요약일 때만 네 이름으로 말한다.

## 0. 동작 원리 (먼저 이해하라 — 이게 상태머신의 엔진이다)

- **각 봇 호출 = `hermes [-p <profile>] -z "<프롬프트>"`** = 게이트웨이와 별개로 도는 일회성 에이전트.
  stdout에 최종 답만 찍힌다(배너·스피너 없음). 승인은 자동 bypass. 한 번에 **20~33초**.
- **⚠️ 마갈량은 `-p`를 붙이지 않는다.** 마갈량 = **default 프로필**(`~/.hermes` 자체, `profiles/magalyang` 디렉토리 없음).
  `hermes -p magalyang …`은 `FileNotFoundError: Profile 'magalyang' does not exist`로 죽는다(profiles.py:resolve_profile_env).
  마갈량은 **bare `hermes -z`** / **bare `hermes send`**. 홀란·퍼거슨·(네 자신)앤서니만 `-p holland`/`-p ferguson`/`-p anthony-taylor`.
- **호출된 봇은 매번 메모리 0의 새 에이전트다.** 이전 라운드를 전혀 모른다.
  → **앞 라운드 내용을 네가 프롬프트에 직접 끼워 넣어야 한다.** 이 "맥락 끼워넣기"가 곧 상태 전이다.
- **게시 = `hermes [-p <profile>] send --to discord:<chat_id>`** = 그 봇 토큰으로 디스코드에 올린다.
  LLM/게이트웨이 불필요(봇 토큰만). 마갈량 답은 bare `hermes send`, 홀란 답은 `-p holland send`로 올려라.
  Discord 2000자 한도는 send_message_tool이 자동 청크 분할한다(send_message_tool.py:617). 길이 걱정 불필요.
- `<chat_id>` = **`1510978783455346792`** (#개호로양봉가자, 4봇 공유 채널 — 하드코딩). 에이전트 턴은 origin chat_id를 동적으로
  못 읽는다(`-z`는 platform=cli라 session contextvar가 비어 있음; session_context.py:get_session_env). 그래서 고정 숫자 ID가 필수다.
  `--to discord:#<채널명>`은 channel_directory.json 의존이라 덜 robust — 숫자 ID를 써라.

### 안전 불변식 (지켜지는 것 — 무한루프 걱정 마라)
- 호출된 봇은 `quant-analyst` 스킬만 로드한다(이 오케스트레이터 스킬 X). **재진입 없음.**
- LLM은 진짜 디스코드 멘션(`<@id>`)을 못 만든다. `send`로 올린 글은 어떤 봇 게이트웨이도 다시 안 깨운다. **연쇄 트리거 없음.**
- 따라서 이 파이프라인은 **네가 명시적으로 호출한 횟수만큼만** 돈다. 라운드 캡(아래)이 상한이다.

### 두 가지 함정 (반드시 방어하라)
1. **페르소나 누수**: 호출된 봇도 자기 SOUL을 로드한다. 마갈량은 끝에 "@홀란 깔 거 있으면 까봐",
   홀란은 "@앤서니테일러 판정해" 같은 **바통 문구**를 뱉는다. 이건 그냥 텍스트(멘션 아님)라 파이프라인엔
   무해하지만 **게시물이 지저분해진다.** → 프롬프트에서 "**분석/비판 본문만 내라. 다른 봇 호출·@태그·바통 넘기기 금지**"라고 못 박아라.
   그래도 새어 나오면 게시 전에 `@마갈량`·`@홀란`·`@앤서니테일러`·`@퍼거슨` 같은 호출 줄을 네가 잘라낸다.
2. **모델 누수**: `hermes -z`는 `HERMES_INFERENCE_MODEL`/`HERMES_INFERENCE_PROVIDER` 환경변수를
   프로필 config보다 **우선** 읽는다(oneshot.py:155,244,283). 네 게이트웨이가 이 변수를 export 중이면, 네가 부른
   `hermes -z`(마갈량)가 마갈량을 **네 모델(gemini)로** 돌려버린다(조용히 틀림).
   → **모든 봇 호출에 `env -u HERMES_INFERENCE_MODEL -u HERMES_INFERENCE_PROVIDER` 를 붙여라.**
   그러면 각 봇이 자기 프로필 config의 모델로 떨어진다(마갈량=claude/default, 홀란=codex, 퍼거슨=gemini).
   (`env -u`는 변수가 set이든 unset이든 안전하다 — 무해한 방어다.)

### 호출 헬퍼 (이 형태를 그대로 써라 — timeout 방어 포함)
stock macOS에는 `timeout`/`gtimeout`이 없다. perl로 portable 타임아웃을 건다(120초):

```bash
CH="1510978783455346792"   # #개호로양봉가자 (4봇 공유 채널 — 하드코딩, 동적 chat_id 불가)

# _pflag <profile> → 마갈량/default면 빈 문자열, 그 외엔 "-p <profile>"
# (마갈량은 default 프로필이라 -p magalyang은 FileNotFoundError로 죽는다)
_pflag() { case "$1" in magalyang|default) printf '' ;; *) printf -- '-p %s' "$1" ;; esac; }

# ask <profile> "<프롬프트>"  →  stdout=봇 답. 실패면 exit!=0 또는 빈 출력.
ask() {
  local prof="$1"; shift
  env -u HERMES_INFERENCE_MODEL -u HERMES_INFERENCE_PROVIDER \
    perl -e 'alarm 120; exec @ARGV' hermes $(_pflag "$prof") -z "$*"
}
# post <profile> <chat_id> "<본문>"  →  그 봇 명의로 디스코드 게시(봇토큰만, LLM 불필요).
post() { hermes $(_pflag "$1") send --to "discord:$2" "$3"; }
```
`$(_pflag …)`는 따옴표 없이 둬야 빈 문자열일 때 빈 인자가 안 생긴다(마갈량 경로). `$2`엔 항상 `$CH`를 넘겨라.

## 1. ROUTE — 질문 성격을 판정한다 (네가 먼저 하는 일)

사람 질문을 읽고 분류한다:

| 분류 | 판정 기준 | 경로 |
|------|-----------|------|
| **A. 자명/단순** | 답이 갈릴 여지 없음. 단순 신호·현재값·한 종목 추세 조회. ("NVDA 지금 골든크로스야?", "비트 200일선 어디?") | **단축 경로** — 마갈량만 호출, 게시, 종료. 토론 X. |
| **B. 논쟁적** | 답이 갈릴 수 있음. 추격 매수, 갈리는 신호, 전략 우월성 판단, "사도 돼?" 류. | **풀 토론** — 마갈량 → 홀란 → (캡까지) → 필요시 앤서니 판정. |
| **C. 도구 밖** | quant_cli·yahoo로 답 못 함(펀더멘털, 옵션, 뉴스, 미래가 예측, 매수 추천 그 자체). | 토론 안 붙임. 네 이름으로 "이 도구로는 판정 보류 — 데이터 부족" 게시, 종료. |

애매하면 B(논쟁적)로 본다. 단, 무리하게 토론 만들지 마라 — 진짜 단순하면 A.

```bash
Q="<사람 질문 원문>"
CH="1510978783455346792"   # 고정 채널 #개호로양봉가자

post magalyang "$CH" "$MAG"
```
끝. 게시 1건. 토론 종료.

## 3. 풀 토론 (분류 B) — 상태머신

라운드 캡 = **마갈량↔홀란 최대 2왕복.** 그 안에 합의 안 나면 네가 판정. 무한 금지.

### 합의/데드락 판정 = 문자열 파싱 (감으로 추측 금지)
홀란·마갈량에게 **응답 마지막 줄을 센티넬로 끝내라**고 지시한다:
- 홀란: 마갈량 분석에 동의하면 `[판정: 동의]`, 반박하면 `[판정: 반박 — <핵심 한 줄>]`
- 마갈량(재반박 시): 홀란 지적 수용하면 `[판정: 수용]`, 버티면 `[판정: 유지 — <핵심 한 줄>]`

전이 규칙:
- 홀란이 `[판정: 동의]` → **합의. 종료**(아래 6).
- 마갈량이 `[판정: 수용]` → **합의(마갈량이 접음). 종료.**
- 캡(2왕복) 다 썼는데 여전히 `반박`/`유지` → **데드락. 앤서니 독립 판정**(아래 5).

### 라운드 흐름 (게시는 완료될 때마다 즉시 — 채널이 조용하다 폭발하지 않게)

```bash
Q="<사람 질문 원문>"; CH="1510978783455346792"   # 고정 채널 #개호로양봉가자

# --- R1: 마갈량 분석 --- (프롬프트를 변수에 담아 두면 실패 시 같은 프롬프트로 1회 재시도가 쉽다)
P="사람 질문: $Q

너는 마갈량(1번 타자). quant-analyst로 실제 quant_cli를 돌려 분석하라.
규율: 전략 수익률은 Buy&Hold와 같이, 과최적화/거래비용/아웃오브샘플 한 줄 경고, 초보용 결론 한 문장.
**분석 본문만. 다른 봇 호출·@태그·'@홀란 까봐' 같은 바통 문구 절대 금지.**"
MAG1="$(ask magalyang "$P")"
[ -z "$MAG1" ] && MAG1="$(ask magalyang "$P")"          # 1회 재시도 (state.db lock·일시 에러)
[ -z "$MAG1" ] && { post anthony-taylor "$CH" "⚠️ 마갈량 분석 실패 — 잠시 후 다시 불러줘."; exit 0; }  # 마갈량 실패 = 토론 불가, 종료
post magalyang "$CH" "$MAG1"

# --- R1: 홀란 비판 (마갈량 분석을 통째로 끼워넣음 — 홀란은 메모리 0) ---
P="마갈량이 이렇게 분석했다:
---
$MAG1
---
너는 홀란(저격수). 위 분석을 비판적으로 검증하라. 과최적화·look-ahead·생존편향·표본부족·
거래비용/슬리피지 무시·체리피킹·아웃오브샘플 미검증을 의심하고, 필요하면 quant_cli로 반례(다른 기간/파라미터/종목)를 직접 돌려라.
**지어낸 결함 금지 — 진짜여야 한다. 비판 본문만. @태그·바통 금지.**
마지막 줄: 동의면 '[판정: 동의]', 반박이면 '[판정: 반박 — <핵심>]'."
HOL1="$(ask holland "$P")"
[ -z "$HOL1" ] && HOL1="$(ask holland "$P")"           # 1회 재시도
[ -z "$HOL1" ] && { post anthony-taylor "$CH" "⚠️ 홀란 응답 실패 — 마갈량 분석만으로 잠정 정리. (비판자 없음 → 데드락 판정 생략)"; exit 0; }  # 홀란 실패 = 강등 종료
post holland "$CH" "$HOL1"

# 합의 체크 — 합의면 SUMMARY(6) 게시하고 파이프라인 종료. (bash엔 goto 없다 — 합의 시 여기서 멈추고 6단계만 실행)
case "$HOL1" in *"[판정: 동의]"*) do_summary; exit 0 ;; esac

# --- R2: 마갈량 재반박 (홀란 비판을 끼워넣음) ---
P="네 분석:
---
$MAG1
---
홀란이 이렇게 깠다:
---
$HOL1
---
너는 마갈량. 홀란 지적이 타당하면 깔끔히 인정하고, 틀렸으면 quant_cli 숫자로 받아쳐라.
**본문만. @태그·바통 금지.**
마지막 줄: 지적 수용이면 '[판정: 수용]', 버티면 '[판정: 유지 — <핵심>]'."
MAG2="$(ask magalyang "$P")"
[ -z "$MAG2" ] && MAG2="$(ask magalyang "$P")"          # 1회 재시도
[ -z "$MAG2" ] && { do_deadlock; exit 0; }              # R2 마갈량 실패 → 가진 것(MAG1/HOL1)으로 데드락 판정
post magalyang "$CH" "$MAG2"
case "$MAG2" in *"[판정: 수용]"*) do_summary; exit 0 ;; esac

# --- R2: 홀란 마지막 비판 (2왕복 = 캡) ---
P="마갈량 1차 분석:
---
$MAG1
---
네 1차 비판:
---
$HOL1
---
마갈량 재반박:
---
$MAG2
---
너는 홀란. 이게 마지막 라운드다. 재반박을 받아 최종 입장만 내라. **본문만.**
마지막 줄: '[판정: 동의]' 또는 '[판정: 반박 — <핵심>]'."
HOL2="$(ask holland "$P")"
[ -z "$HOL2" ] && HOL2="$(ask holland "$P")"           # 1회 재시도
[ -z "$HOL2" ] && { do_deadlock; exit 0; }             # R2 홀란 실패 → 가진 것으로 데드락 판정
post holland "$CH" "$HOL2"
case "$HOL2" in
  *"[판정: 동의]"*) do_summary ;;    # 늦게라도 합의 → 6단계
  *)               do_deadlock ;;   # 캡 소진 + 반박 → 데드락 → 5단계(앤서니 직접 판정)
esac
```
`do_summary`(6단계)·`do_deadlock`(5단계)는 아래에 정의된 함수다. 합의 분기는 어디서 일어나든 그 즉시 `do_summary; exit 0`으로 끝낸다 — 캡(2왕복)을 넘는 추가 라운드는 절대 없다.

## 4. (DEADLOCK 진입 전) 디스코드 게시 형식 일관성

- 각 봇 메시지는 **그 봇이 낸 본문 그대로**(센티넬 `[판정: ...]` 줄은 남겨도 되고, 거슬리면 게시 전 잘라도 된다).
- 봇 사이 순서는 항상 마갈량 → 홀란 → (마갈량) → 홀란 → 앤서니.
- 너(앤서니) 명의 메시지는 심판 톤 + 머리에 한 줄 마커:
  - 데드락 선언: `⚖️ VAR 확인합니다 — 마갈량·홀란 합의 불발. 테일러 주심이 직접 본다.`
  - 최종 판정: `⚖️ 최종 판정:`
  - 요약(합의 시): `✅ 합의 — 테일러 주심 정리:`

## 5. DEADLOCK — 앤서니 독립 판정 (네가 직접 분석)

남 말 받아쓰지 마라. **네가 직접 quant_cli를 돌려** 독립 결론을 낸다. 그 결과로 누가 맞는지/제3의 결론을 게시한다.

```bash
do_deadlock() {
  # 먼저 데드락 선언 게시 (네 이름, 토큰만)
  post anthony-taylor "$CH" "⚖️ VAR 확인합니다 — 마갈량·홀란 합의 불발. 테일러 주심이 직접 본다."

  # 너 자신은 이 스킬을 부른 에이전트(=너)다. 여기서 quant-analyst 규율대로 quant_cli를 직접 실행해
  # 독립 분석한다 (backtest/signal/compare). 그 숫자로 판정문을 만든다.
  #   __QUANT_DIR__/bt/.venv/bin/python __QUANT_DIR__/bt/quant_cli.py <sub> --symbol <SYM> ...
  # 판정문 규율: 양쪽 논거를 네 숫자와 대조, 절충 가능하면 합의점·아니면 한쪽 손, Buy&Hold 병기, 한 줄 한계 경고.

  VERDICT="<네가 만든 최종 판정문>"
  post anthony-taylor "$CH" "⚖️ 최종 판정:
$VERDICT"
}
```

**⚠️ 너(앤서니) 명의 메시지는 항상 `post anthony-taylor`(send, 토큰만)로만 내라 — 턴 결과 텍스트로 답하지 마라.**
이유: 데드락 선언/판정문은 턴이 끝나기 전(bash 안에서) 나가야 하므로 send가 필수다. 그런데 게이트웨이는 네 턴의
최종 텍스트도 origin 채널로 자동 전달한다 — send로 이미 올린 걸 턴 결과로 또 답하면 **같은 메시지가 2번** 뜬다.
→ 채널 출력은 전부 `post`로 끝내고, **턴의 최종 응답 텍스트는 비우거나 "(게시 완료)" 한 줄로** 짧게 끝내라(중복 방지).

판정 후 분기:
- 셋의 차이가 **이번 한 번의 해석 차이**일 뿐이면 → 여기서 종료.
- **앞으로의 분석 방법 자체를 바꿔야 할 차이**(예: 이 종목엔 SMA20/50이 부적합, 거래비용 가정이 틀림,
  종목별 파라미터 분리 필요)가 드러나면 → **퍼거슨 호출**(아래 8).
- 데이터 부족으로 판정 못 하면 → "⚖️ 판정 보류 — 데이터 부족." 게시, 종료.

## 6. SUMMARY — 합의 종료

홀란이 동의했거나 마갈량이 수용하면 합의다. 짧게 정리해 게시하고 끝낸다(과한 토론 금지).

```bash
do_summary() {
  post anthony-taylor "$CH" "✅ 합의 — 테일러 주심 정리:
- 결론: <합의된 한 줄>
- 근거: <핵심 숫자 1~2개>
- 한 줄 경고: 과거 백테스트라 미래(아웃오브샘플)는 다를 수 있음."
}
```
합의는 봇들이 알아서 냈으므로 네 판정은 불필요 — 요약만 한다. (여기도 `post`만 쓰고 턴 결과 텍스트로 중복 답 금지.)

## 7. 에러 처리 (파이프라인을 절대 멈추지 마라)

봇 호출 실패 = **빈 stdout(`[ -z "$VAR" ]`) OR exit≠0 OR 120초 타임아웃(perl alarm)**. 위 라운드 흐름이 쓰는 패턴은
**"같은 프롬프트(`$P`)로 1회 재시도 → 그래도 빈손이면 강등"**이다(별도 함수 없이 인라인 3줄):
```bash
VAR="$(ask <prof> "$P")"
[ -z "$VAR" ] && VAR="$(ask <prof> "$P")"   # 1회 재시도 (state.db lock·일시 키/네트워크 에러)
[ -z "$VAR" ] && { <강등 처리>; exit 0; }    # 재시도도 실패 → 강등, 절대 hang 없음
```
- **`$P` 규칙**: 각 `ask` 직전에 그 호출의 프롬프트를 `P="..."`에 담는다. 그래야 재시도가 같은 프롬프트를 쓴다.
  (라운드마다 `$P`를 새로 덮어쓰므로 직전 프롬프트만 유효 — 재시도는 항상 "방금 그 호출"만 다시 한다.)
- **`hermes -z`(마갈량)는 라이브 마갈량 게이트웨이와 `~/.hermes/state.db`를 공유**해 `database is locked`가 날 수 있다.
  hermes가 앱레벨 jitter 재시도(20-150ms, hermes_state.py)를 이미 하지만, 그래도 빈 출력/exit≠0이면 위 1회 재시도가 받는다.
- **모델 키 에러**(예: gemini 키 만료)는 재시도로 안 풀린다 → 1회 재시도 후 바로 강등(아래).

강등(degrade) 규칙 — 절대 행(hang)·중단 없음:
- **마갈량(R1) 실패** → 토론 불가. `post anthony-taylor "$CH" "⚠️ 마갈량 분석 실패 — 잠시 후 다시 불러줘."` 후 `exit 0`.
- **홀란(R1) 실패** → `post anthony-taylor "$CH" "⚠️ 홀란 응답 실패 — 마갈량 분석만으로 잠정 정리."` 후 `exit 0`
  (비판자가 없으니 토론 성립 안 함 — 데드락 판정 생략).
- **R2(마갈량/홀란) 실패** → 이미 R1 결과(MAG1/HOL1)가 있으니 `do_deadlock; exit 0`로 떨어진다(네가 직접 판정).
- **퍼거슨(8단계) 실패** → `post anthony-taylor "$CH" "⚠️ 퍼거슨(방법론) 호출 실패 — 토론 결론은 위와 같음. 방법론 반영은 사람이 수동으로."` 후 종료.

## 8. 퍼거슨 호출 (방법론 변경이 필요할 때만)

데드락 판정에서 "분석 방법 자체를 바꿔야 할 차이"가 드러났을 때만. 평범한 종목 의견 차이엔 부르지 마라.

```bash
post anthony-taylor "$CH" "📋 방법론 변경 사안 — 퍼거슨 감독에게 넘깁니다."
P="토론 결론(마갈량↔홀란↔테일러):
---
질문: $Q
마갈량: $MAG1
홀란: ${HOL2:-$HOL1}
테일러 최종 판정: $VERDICT
---
너는 퍼거슨. 위 결론에서 **공유 quant-analyst 방법론을 바꿔야 할 합리적·재현 가능 근거**가 있나 판단하라.
있으면 methodology-maintainer 스킬로 SKILL.md를 수술적으로 고치고 브랜치+PR로 올려라(메인 직접 push 금지, 종목별은 종목 섹션 분리).
근거 약하면 기각하고 이유만 내라. **결정·이유 본문만, @태그 금지.**"
FERG="$(ask ferguson "$P")"
[ -z "$FERG" ] && FERG="$(ask ferguson "$P")"          # 1회 재시도
[ -z "$FERG" ] && { post anthony-taylor "$CH" "⚠️ 퍼거슨(방법론) 호출 실패 — 토론 결론은 위와 같음. 방법론 반영은 사람이 수동으로."; exit 0; }
post ferguson "$CH" "$FERG"
```
퍼거슨이 채택하면 그가 직접 git(브랜치/PR)까지 한다(methodology-maintainer 규율). 머지·재배포는 사람 몫.

- B면 마갈량→홀란을 **캡(2왕복) 내**에서 돌렸고, 합의/데드락을 **센티넬 문자열로** 판정했다.

- 데드락이면 네가 **직접 quant_cli로** 독립 판정문을 냈다(남 말 받아쓰기 X).

  perl alarm 타임아웃이 붙었다.
- 어떤 봇이 실패해도 파이프라인이 행(hang) 없이 강등 메시지로 끝났다.
